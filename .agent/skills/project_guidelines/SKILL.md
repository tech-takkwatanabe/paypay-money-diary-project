# Project Guidelines: Architecture & Coding Standards

Source of truth: [`PROJECT_RULES.md`](../../../PROJECT_RULES.md)

## 1. Architecture

- **Monorepo**: Turborepo + pnpm
- **Frontend** (`apps/web`): Next.js App Router, Tailwind CSS, Orval
- **Backend** (`apps/api`): Hono, Drizzle ORM, PostgreSQL
- **Shared** (`packages/shared`): Types, Zod Schemas, Utils
- **Testing**: Vitest / Testing Library

## 2. Directory Structure

### `apps/api/src`

`config`, `controllers`, `domain` (`dto`, `entity`), `usecase`, `service`, `models`, `infrastructure`, `middlewares`, `utils`, `docs`, `docker`

### `apps/web/src`

`api` (generated by Orval), `components`, `providers`, `stories`, `utils`, `app` (App Router)

### `packages/shared`

`types` (Zod schemas, `vo/` for Value Objects), `utils`

## 3. Coding Standards

### Naming Conventions

| Target | Convention | Example |
|---|---|---|
| Variables / Functions | `camelCase` | `formatDate` |
| Component Files | `PascalCase` | `TransactionList.tsx` |
| Hooks / Utils | `camelCase` | `useUser.ts` |
| Zod Schemas | `camelCase` + `.schema.ts` | `transaction.schema.ts` |
| Constants | `UPPER_SNAKE_CASE` | `MAX_ROWS` |
| DB Tables | `snake_case` | `expense_categories` |

### Code Style

- **`any` 型の使用禁止**: 型ガード (`instanceof Error` など) を使用する
- **Tailwind CSS**:
  - ユーティリティクラスは必ず JSX の `className` 内に記述
  - **禁止**: `clsx`, `cva`, `cn` などのライブラリ
  - 条件分岐は `className` 内で完結させる（例: `condition ? "text-red-500" : "text-blue-500"`）
- **Zod**: API レスポンス、フォーム、共有ドメインモデルのバリデーションに使用
  - API 入出力には必ず Zod バリデーションを適用すること
  - 意味のあるバリデーションを含むこと（形だけのスキーマは不可）
- **Components**: Functional + Hooks。小さく焦点を絞る（SRP）
- Server Components を優先（クライアント状態が必要な場合のみ `"use client"`）
